# Dynamic Island + Dotmatrix Loaders + Voice Dictation

Status: planning (verified against codebase + upstream/main @ `6d4ae914` + dotmatrix registry JSON + dotmatrix GitHub repo + whisper.cpp v1.9.1 + build config interface). All research done. One integrated pass, three subsystems.

## Goals

1. **Dotmatrix loaders** replace the sidebar spinner and power the island's per-state loader. User-picked: Echo Ring (thinking), Core Rotor (long command), Half Helix (editing), Origin Wave (reading). Colors are per-action (not per-provider), user picks from preset themes.
2. **Dynamic island** below the top bar: compact pill when an agent is running (current action text + small state loader), hidden when idle (hover-to-reveal shows last activity + recent threads), hover-expand to a panel with more detail + mini chat. Auto-expands on approval/user-input/plan-proposed. Pill on all platforms (mac pill appears below the notch because the 46px header clears it).
3. **Voice dictation** â€” two surfaces, same engines:
   - **In-composer mic button** (existing, currently Codex-only): make it universal. Codex keeps ChatGPT endpoint (untouched). All other providers route through local whisper.cpp.
   - **New floating popup** (willow/superwhisper style, hold right-Option): available for all providers. Codexâ†’ChatGPT, othersâ†’whisper. Edit transcript before send. Custom dictionary.

## Decisions locked (all clarifying rounds)

### Platform + scope
- **Platform**: pill below the top bar on all platforms. Same component, adaptive shape. Position: `top: calc(46px + 4px)`, centered. On mac-with-notch, the 46px header already clears the notch, so the pill appears immediately below the notch â€” the "dynamic island" aesthetic without notch IPC. Idle hover-to-reveal works everywhere (tiny pill is always visible).
- **Browser voice**: desktop only. In the browser, the mic button IS visible for all providers (because `voiceTranscriptionAvailable === undefined` passes `!== false`). Clicking it for Codex works (server-side `wsRpc.ts` routes to Codex adapter). Clicking it for non-Codex fails with "Voice transcription is unavailable for provider 'X'" (server-side `providerAdapterRegistry` has no `transcribeVoice` adapter for non-Codex). Ponytail: let the error happen â€” no pre-check needed, the error message is clear. The popup (hold right-Option) is desktop-only â€” the hotkey listener is registered only when `window.desktopBridge` is available.
- **Multi-window**: each window mounts its own island, hidden when not focused via `document.hasFocus()` + `window.blur` event. Effectively a global single island â€” only the focused window shows it, zero cross-window coordination needed. Each window independently receives all thread events via its own WS, so all windows compute the same most-recently-active thread. When the app is not focused (you're in another app), all windows hide their islands.
- **Island z-index**: `z-50` (above content, below desktop window controls at `z-[250]`).

### Island behavior
- **Thread source (TWO modes)**: (1) **active mode** = most-recently-active thread that is running OR has pending state (`thread.hasPendingApprovals === true` / `thread.hasPendingUserInput === true` / `thread.hasActionableProposedPlan === true` â€” all OPTIONAL fields, use `=== true` not truthy). Used for compact + auto-expand. (2) **idle mode** = most-recently-active thread by `thread.updatedAt` (OPTIONAL â€” fall back to `thread.activities?.at(-1)?.createdAt ?? thread.createdAt`) or `thread.lastVisitedAt` (OPTIONAL) regardless of running/pending state. Used for idle hover (shows last activity from ANY thread, not just running ones). 150ms hysteresis on mode 1 to prevent flicker. Mode 1 ensures the island stays visible when an agent proposes a plan or requests approval and the turn has ended. "Most-recently-active" = max of `thread.latestTurn?.startedAt` (can be `null` â€” `Schema.NullOr(IsoDateTime)`, `orchestration.ts:519`) OR `thread.activities` last entry `createdAt` OR `thread.updatedAt`. Handle `null`/`undefined` `startedAt`/`updatedAt` by falling back to the other timestamps.
- **Thread deletion/archival**: if the tracked thread is deleted or archived (`thread.archivedAt` â€” `types.ts:191`, OPTIONAL `string | null`), the tracker picks the next most-recently-active running/pending thread, or hides if none qualify. No stale references.
- **Store selector performance**: `useStore((s) => s.threads)` re-renders on EVERY thread change. The island uses a memoized selector that extracts only the tracked thread (by ID) + the recent threads list (for idle hover). Use `useStore` with `useShallow` or a custom selector that returns a stable reference when the tracked thread hasn't changed.
- **Single agent display**: one agent at a time in compact. No multi-agent stacking. Ponytail: delete the complexity.
- **Idle state**: hidden. Tiny always-visible 4px pill at top-center as the hover affordance (like Boring.Notch's minimal idle state). - **No auto-send**: always manual. Popup appends transcript to composer (preserving existing composer text â€” append, not replace, matching existing Codex voice recorder behavior). User clicks composer send.
- **Concurrent triggers**: shared `voiceDictationStore` (zustand) with `isRecording`/`isTranscribing`/`activeSurface` fields. If any recording is active, block the other surface. Show "Transcribing..." during transcription. Without this store, the in-composer mic and popup can't coordinate. **Cross-window**: the store is per-renderer (zustand is per-window), so two windows can both record simultaneously. With per-request spawn, each transcription gets its own `whisper-cli` process â€” no shared stdin, no corruption possible, no cross-window mutex/queue needed. The ChatGPT path also spawns per-request. Two simultaneous transcriptions = two processes, fully isolated.
- **Desktop only**: no Web Speech API fallback for browser.
- **Max recording duration**: 120s (`MAX_VOICE_DURATION_MS = 120_000` in `voiceTranscription.ts:20`). A 120s recording at 16kHz = 3.84MB WAV (fine for stdin). Transcription time at ~7x realtime = ~17s. The popup inherits this limit (same recorder). Show a countdown or "max 120s" hint in the popup UI.

### 3. Abuse / security

**Voice audio**: `getUserMedia` (user-initiated, permission required). Local whisper sidecar â€” audio never leaves machine for non-Codex. Codex path to ChatGPT unchanged.

**Custom dictionary**: `voiceDictionary: string[]` in local settings, capped at 200 entries. Prepended to whisper prompt. No injection â€” whisper prompt is a model string, not a shell command. User-controlled local data, no trust boundary. Cap prevents accuracy degradation from oversized prompts.

**Island mini chat**: `dispatchCommand` â€” same trust as main composer. No attachments/mentions in v1. No new attack surface.

**Island approval/user-input buttons**: same APIs as existing composer panels. No new trust boundary.

**Activity label content from provider**: `WorkLogEntry.label` is a free-form string from provider runtime events. A malicious or buggy agent could inject RTL override chars, zero-width chars, or text that looks like a system message. `actionStates.ts` sanitizes labels: truncated to 80 chars, control characters stripped, full text available via `title` attribute. React escapes HTML by default. Visual distinction (icon + color) separates activity labels from system messages.

**whisper.cpp sidecar**: native code in separate process. Malicious model risk â†’ verify SHA256 against hardcoded values in source. Only official ggml HuggingFace or user-specified local path. Sidecar isolation â€” whisper exploit can't touch Electron main process.

**whisper command injection**: `execFile` (not `exec`), args as array (no shell). Model path validated (no `..` traversal, no shell metachars). Audio piped via stdin (`whisper-cli -f -`) â€” no temp file on disk, no path validation needed for audio.

**Notch detection IPC**: dropped â€” no IPC needed. Island positions at `top: calc(46px + 4px)` everywhere. No injection surface.

**Dotmatrix vendored code**: pure React + CSS, no network/eval/dynamic imports (verified by reading source). Low risk. Review before merge.

**Server voice gate change**: **NONE.** The `ProviderHealth.ts` change was removed â€” `resolveVoiceTranscriptionAvailability` is only called for Codex, and non-Codex providers already pass the voice gate (`voiceTranscriptionAvailable === undefined` passes `!== false`). No server-side attack surface expansion. All whisper routing happens in the desktop bridge (local process, no network). The server RPC handler still validates the input shape via the existing Effect Schema for the browser fallback path.

### 4. Scale / performance

**Island render frequency**: `thread.activity-appended` 5-20/sec. Memoize on last entry identity. `domainEventFlushThrottler` coalesces at 100ms (`__root.tsx:1122`). 150ms hysteresis on thread tracking. ~10 label updates/sec max, no thrash.

**Dotmatrix cost**: 5x5 grid (25 elements) per loader, CSS animations, GPU-composited. 1 loader in island compact, 0-1 in sidebar simultaneously. Negligible. Even 20 running threads in sidebar = 500 elements.

**whisper.cpp memory**: base-q5_1 = 57MB disk, ~200-300MB RAM (quantized, less than the ~388MB of non-quantized base). Per-request spawn â€” memory held only during the 5-17s transcription, freed when the process exits. No persistent footprint between dictations. Fine on 8GB+, invisible on 16GB+.

**Model download**: 57MB one-time. Progress bar. `userData/whisper-models/`. Offline on first use â†’ "connect once, then offline forever." Restart from 0 on interruption (v1).

**whisper-cli binary**: **Windows** ~5MB (`whisper-bin-x64.zip`), **Linux** ~13MB (`whisper-bin-ubuntu-x64.tar.gz`), bundled via `extraResources`. **Mac**: NOT bundled â€” detected in PATH at runtime (Homebrew `brew install whisper-cpp`). +5-13MB installer on Windows/Linux only. Acceptable for desktop.

**Multiple running threads**: island tracks one (most-recently-active running or pending). Sidebar shows loaders per running thread. Each 25 DOM elements. 20 threads = 500 elements, negligible.

**Island expanded view**: last 4 activities, no virtualization. Hard cap.

**Approval/user-input auto-expand**: rare (user-initiated tool calls). One-time transition per event. No perf concern.

**Multi-window**: each window has its own island + WS. No shared state. Scales linearly with windows. 2-3 windows typical, negligible.

**24â†’16kHz resampling**: ~30 lines of linear interpolation in the desktop process. For a 60s recording at 24kHz = 1.44M samples â†’ 960k samples at 16kHz. ~10ms on any modern CPU. Negligible.

### 5. Trade-offs

**whisper: sidecar binary vs node addon** â†’ **sidecar** (`whisper-cli`, spawned per-request). **Model: base.en-q5_1 (57MB)** â†’ 2nd smallest. tinyâ†’base halves WER (7.6%â†’5.0%) for +26MB. small is 3x size for marginal gain. Setting allows switching.

**Island shape: pill below the top bar** â†’ not notch-hugging (no clip-path, no notch IPC).
